# 像素画生成器 - 项目结构说明

## 项目概述

这是一个完整的像素画生成器项目，实现了 PRD 文档中描述的所有核心功能。项目采用 Node.js 全栈开发，包含前端界面和后端图像处理服务。

## 文件结构

```
pixel-art-generator/
├── server.js                 # 主服务器文件
├── package.json             # 项目配置和依赖
├── README.md               # 项目说明文档
├── deploy.bat              # Windows 部署脚本
├── PROJECT_INFO.md         # 项目结构说明（当前文件）
├── public/                 # 前端资源目录
│   ├── index.html          # 主页面
│   ├── style.css           # 样式文件
│   └── script.js           # 前端逻辑
└── temp/                   # 临时文件目录（运行时创建）
```

## 功能实现详情

### 1. 前端功能
- **图片上传模块**：支持拖拽上传和点击选择上传
- **参数自定义配置面板**：色彩位数和像素块大小选择
- **实时预览与对比展示**：双栏显示原图和生成结果
- **结果操作功能**：下载生成的像素画

### 2. 后端功能
- **图片处理接口**：基于 Sharp 库的像素化处理
- **文件与安全处理**：上传校验和格式检查
- **性能优化**：高效的图像处理算法

### 3. 用户体验
- **响应式设计**：适配桌面和移动设备
- **加载状态**：处理过程中的加载动画
- **错误处理**：友好的错误提示

## 技术实现亮点

1. **像素化算法**：
   - 先将图片按指定像素块大小缩小
   - 使用最近邻插值保持硬边缘效果
   - 按指定色彩数进行量化处理

2. **前后端通信**：
   - 使用 FormData 进行文件上传
   - 支持实时参数传递
   - 异步处理避免页面阻塞

3. **性能优化**：
   - 内存流式处理，避免大文件占用过多内存
   - 高效的 Sharp 图像处理库

## 部署说明

### 方法1：使用部署脚本（推荐）
- Windows 用户运行 `deploy.bat`
- 脚本会自动检查环境、安装依赖并启动服务器

### 方法2：手动部署
1. 确保已安装 Node.js v14+
2. 运行 `npm install` 安装依赖
3. 运行 `node server.js` 启动服务器
4. 访问 `http://localhost:4000`
