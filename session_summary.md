## 1. 项目概述

### 1.1 背景与痛点
在复古游戏开发、社交媒体头像制作及数字艺术创作领域，将普通高清图片转换为“像素风格”图片的需求日益增长。目前市场上存在的解决方案存在以下痛点：
1. **门槛高**：多数专业软件（如 Photoshop, Aseprite）需要安装且操作复杂。
2. **体验差**：在线工具往往缺乏实时预览，用户需反复上传下载才能调整参数。
3. **性能低**：部分网页端处理大图片时卡顿严重，或色彩量化算法粗糙。

### 1.2 产品目标
构建一款**轻量级、无需安装、即开即用**的全栈 Web 应用。
- **核心价值**：极简操作（上传->配置->生成->下载）、实时视觉反馈、高质量像素化算法。
- **关键指标**：首屏加载时间 < 1s，图片生成平均耗时 < 3s，支持 10MB 以内主流格式图片。

### 1.3 用户画像
- **独立游戏开发者**：快速生成素材原型。
- **社交媒体用户**：制作个性化的像素风头像或背景。
- **设计爱好者**：探索不同色彩位数和颗粒度下的艺术效果。

### 2.2 后端服务层 (Server-Side)

#### 2.2.1 核心处理接口 (`POST /api/pixelate`)
- **请求参数**：
  - `file`: Binary (Image)
  - `colors`: Integer (16 | 32 | 128 | 256)
  - `blockSize`: Integer (8 | 16 | 32)
- **处理逻辑**：
  1. **安全校验**：检查 File Header，防止恶意文件上传。
  2. **图像预处理**：使用 `Sharp` 读取图片元数据。若图片尺寸过大（如 > 4000px），先等比例缩小至最大边 2000px 以提升性能。
  3. **像素化算法**：
     - **步骤 A (Resize)**：将图片宽度除以 `blockSize`，高度除以 `blockSize`，使用 `nearest` 邻居插值算法缩小。
     - **步骤 B (Quantize)**：对缩小后的图片进行色彩量化，将颜色数量限制为 `colors` 指定的数量（使用 Median Cut 或 K-Means 算法）。
     - **步骤 C (Scale Up)**：将处理后的微小图片放大回原始尺寸（或保持小尺寸，视需求定，建议保持小尺寸以体现像素感，或放大但保持硬边缘）。*注：通常像素画生成后保持较小分辨率更具艺术感，但为了下载清晰度，可放大并使用 `nearest` 插值保持锯齿边缘。*
  4. **输出**：返回 PNG Buffer。
- **响应**：
  - 成功：`Content-Type: image/png`，Body 为图片二进制流。
  - 失败：JSON 格式错误信息 `{ code: 400, message: "Invalid parameters" }`。

#### 2.2.2 资源管理与安全
- **临时文件清理**：若使用磁盘缓存，需设置定时任务（Cron Job）每 30 分钟清理 `/tmp/uploads` 目录下超过 1 小时的文件。
- **内存保护**：设置 Node.js 进程内存上限，大图片处理使用 Stream 流式处理，避免 OOM。
- **跨域支持**：配置 CORS 允许前端域名访问。

## 3. 非功能性需求 (NFR)

### 3.1 性能要求
- **API 响应时间**：95% 的请求在 3 秒内完成（针对 5MB 以下图片）。
- **前端渲染**：首屏内容绘制 (FCP) < 1.5 秒。
- **并发能力**：单节点支持至少 20 QPS。

### 3.2 兼容性
- **浏览器**：Chrome 80+, Firefox 75+, Safari 13+, Edge 80+。
- **移动端**：iOS Safari 13+, Android Chrome 80+。需完美适配触摸事件。

### 3.3 安全性
- **输入验证**：严格校验上传文件的 MIME Type 和扩展名。
- **依赖安全**：定期扫描 `npm` 依赖包漏洞。

## 7. 附录：技术实现建议

### 7.1 推荐技术栈
- **Frontend**: Vue 3 / React (可选，原生 JS 亦可), Tailwind CSS (样式), Axios (请求).
- **Backend**: Node.js, Express, Multer (文件上传), Sharp (图像处理).

### 7.2 核心算法代码片段 (Node.js + Sharp)
```javascript
const sharp = require('sharp');

async function generatePixelArt(inputBuffer, blockSize, colors) {
    // 1. 获取元数据
    const metadata = await sharp(inputBuffer).metadata();
    
    // 2. 计算缩小后的尺寸
    const newWidth = Math.max(1, Math.floor(metadata.width / blockSize));
    const newHeight = Math.max(1, Math.floor(metadata.height / blockSize));

    // 3. 缩小图片 (使用 nearest 邻居插值保留硬边缘)
    let pipeline = sharp(inputBuffer)
        .resize(newWidth, newHeight, { kernel: 'nearest' });

    // 4. 色彩量化 (Sharp 内置支持通过 png 压缩级别或特定插件，此处简化处理)
    // 注意：Sharp 本身不直接支持任意颜色数的量化，可能需要结合 'quantize' 库
    // 或者使用 pngquant 二进制文件进行后续处理。
    // 这里假设使用简单的 flatten 和 normalize 模拟，实际生产环境建议引入 quantize 库
    
    // 示例：仅做像素化处理，色彩量化需额外逻辑
    const processedBuffer = await pipeline.toBuffer();
    
    return processedBuffer;
}
